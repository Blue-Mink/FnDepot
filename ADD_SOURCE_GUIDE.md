# 飞牛OS：添加 Blue-Mink FnDepot 源并安装敲门knock

本文档演示如何将 Blue-Mink 的 FnDepot 仓库添加到飞牛OS应用中心，并安装敲门knock v2.0.10。

## 准备工作

- 飞牛OS 已安装并运行
- 已安装 FnDepot 客户端（如未安装，可从官方源或任意第三方源搜索「FnDepot」安装）

## 步骤一：打开 FnDepot 源管理

1. 在飞牛OS 桌面或应用列表中找到 **FnDepot** 图标，打开应用
2. 点击右上角 **设置** 齿轮图标
3. 选择 **源管理**

## 步骤二：添加自定义源

1. 在源管理页面点击 **添加源**
2. 将以下地址粘贴到输入框：

   ```
   https://github.com/Blue-Mink/FnDepot
   ```

3. 点击 **确定** 或 **保存**

   > 如果访问 GitHub 较慢，可同时在「加速配置」中添加镜像，例如：
   > ```
   > https://edgeone.gh-proxy.org
   > ```

## 步骤三：同步应用列表

返回 FnDepot 首页，点击顶部 **同步** 按钮（环形箭头图标）。等待同步完成，进度条消失。

## 步骤四：搜索并安装敲门knock

1. 在首页搜索框输入 **敲门knock** 或 **fn-knock**
2. 点击应用卡片进入详情页
3. 检查版本应为 **2.0.10**（支持 amd64 与 arm64）
4. 点击 **安装**，按提示完成安装

## 步骤五：验证安装

安装完成后：

- 应用卡片状态会变为「运行中」或「已安装」
- 你可以在「我的应用」中找到敲门knock
- 首次运行前，建议在应用内配置管理密码与网络访问规则

## 常见问题

### Q1：添加源后只显示一个源或同步失败
- 检查网络：确保系统可访问 GitHub
- 使用加速镜像源并重试同步
- 确认源地址完整无误（不含多余空格或路径）

### Q2：安装时提示「不支持当前架构」
- 确认你的飞牛OS设备架构（x86_64 或 ARM64）
- v2.0.10 同时支持两者，如仍报错请重登或检查 FnDepot 客户端版本

### Q3：如何更新到最新版？
- 从 v2.0.10 起，fn-knock 已内置应用内更新功能
- 在应用设置中开启「自动检查更新」即可

## 回退与卸载

如需移除该源：

1. 进入 **源管理**
2. 找到 `https://github.com/Blue-Mink/FnDepot`
3. 点击右侧 **删除** 或 **移除**
4. 重新同步

卸载敲门knock：

- 在「我的应用」中找到对应卡片，点击 **卸载**
